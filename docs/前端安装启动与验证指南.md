# 前端安装启动与验证指南

> 适用版本：MVP v0.1（React + Vite + Ant Design）
> 后端对应版本：FastAPI，OpenAI 兼容接口层（`/v1/chat/completions`、`/v1/models`）

---

## 一、环境前置要求

| 工具 | 版本要求 | 检查命令 |
|------|---------|---------|
| Node.js | >= 18 | `node -v` |
| npm | >= 9 | `npm -v` |
| Python | >= 3.10 | `python --version` |
| Docker Desktop | 任意最新版 | `docker info` |

---

## 二、后端服务启动（前端依赖）

前端通过 Vite dev proxy 将 `/api` 和 `/v1` 请求转发到 `http://localhost:8000`，因此需要先保证后端正常运行。

### 2.1 启动基础设施

```bash
# 项目根目录
cd AgentBasePlatform

# 启动 PostgreSQL 和 Redis
docker compose up -d postgres redis
```

验证：

```bash
docker compose ps   # postgres / redis 均应显示 running
```

### 2.2 初始化数据库（首次运行）

```bash
# 激活 Python 虚拟环境
source .venv/bin/activate       # macOS / Linux
# .venv\Scripts\activate        # Windows

# 创建数据表
python -m scripts.init_db
```

### 2.3 启动后端 API 服务

```bash
# 项目根目录，激活 venv 后
uvicorn src.main:app --reload --port 8000
```

服务就绪标志：控制台出现 `Uvicorn running on http://0.0.0.0:8000`

> 快速检验：访问 [http://localhost:8000/docs](http://localhost:8000/docs) 应出现 Swagger UI。

## 三、前端安装与启动

### 3.1 安装依赖

```bash
# 进入前端目录
cd frontend

# 安装 npm 依赖（首次或 package.json 有变更时运行）
npm install
```

依赖安装完成后，`frontend/node_modules/` 目录会生成。

### 3.2 启动开发服务器

```bash
# 在 frontend/ 目录下
npm run dev
```

预期输出：

```
  VITE v6.x.x  ready in xxx ms

  ➜  Local:   http://localhost:3000/
  ➜  Network: use --host to expose
```

浏览器打开 [http://localhost:3000](http://localhost:3000) 即可访问前端页面。

### 3.3 端口代理说明

Vite dev server 已配置以下代理（见 `frontend/vite.config.ts`）：

| 前缀 | 转发目标 |
|------|---------|
| `/api` | `http://localhost:8000` |
| `/v1` | `http://localhost:8000` |

前端所有 API 请求无需跨域配置，统一通过代理访问后端。

## 四、页面功能说明

| 路由 | 页面 | 说明 |
|------|------|------|
| `/login` | 登录 / 注册 | 支持账号注册和登录，Token 持久化到 localStorage |

## 五、手动验证流程

按照以下步骤在浏览器中完整走通核心功能：

### 步骤 1：注册账号

1. 打开 [http://localhost:3000/login](http://localhost:3000/login)
2. 切换到「注册」标签页
3. 填写用户名、邮箱、密码，点击「注册」
4. 注册成功后自动跳转到智能体列表页

### 步骤 2：查看智能体列表

1. 登录后自动进入 `/agents`
2. 页面应展示已创建的智能体卡片（类型、状态、模型名称）
3. 若列表为空，可通过 API 或后续步骤创建测试智能体

### 步骤 3：发起对话

1. 点击某个智能体卡片上的「开始对话」按钮
2. 跳转到 `/chat/:agentId`
3. 在底部输入框输入消息，点击发送（或按 Enter）
4. 观察消息是否以流式打字机效果逐字出现
5. AI 回复完成后，消息气泡稳定显示

### 步骤 4：验证工具调用展示（ReActAgent）

1. 在对话页与 ReActAgent 类型智能体交互（如：「用 Python 计算 100 的阶乘」）
2. 回复消息下方应出现可展开的「工具调用」面板
3. 展开后显示工具名称、输入参数、执行结果

## 六、自动化验证脚本

项目提供两个后端接口验证脚本，可在启动前端前确认后端接口是否正常：

### 6.1 OpenAI 兼容接口验证（12 项测试）

```bash
# 项目根目录，激活 venv 后
python -m scripts.validate_agent_chat_api
```

测试覆盖范围：

- ChatAgent 非流式 / 流式 / 多轮对话
- ReActAgent 工具调用（Python 执行、文件读取、MCP 工具、Skill 工具）
- `GET /v1/models` 智能体列表格式
- 错误场景优雅降级

全部通过后输出类似：

```
============================================================
 测试结果汇总  通过 12 / 12
============================================================
✅  1. ChatAgent 非流式 chat completions
✅  2. ChatAgent 流式 SSE
...
✅ 12. GET /v1/models 格式验证
```

### 6.2 MVP 端到端验证（含前端 Proxy）

```bash
python -m scripts.validate_mvp
```

该脚本通过前端 dev server 的代理地址（`http://localhost:3000`）发起请求，模拟真实前端访问路径，验证：

- 注册 → 登录 → 创建智能体 → 列出模型 → 非流式对话 → 流式对话

## 八、常见问题排查

| 现象 | 可能原因 | 解决方案 |
|------|---------|---------|
| 登录后跳转失败 / 白屏 | 后端未启动 | 检查 `http://localhost:8000/docs` 是否可访问 |
| `npm install` 报网络错误 | npm registry 访问慢 | `npm install --registry https://registry.npmmirror.com` |
| 对话无响应 / 超时 | LLM API Key 未配置 | 检查 `.env` 中 `DASHSCOPE_API_KEY` 是否已填写 |
| 数据库连接失败 | Docker 未启动 | `open -a Docker` 启动 Docker Desktop，再执行 `docker compose up -d postgres redis` |
| 端口 3000 被占用 | 其他进程 | 修改 `frontend/vite.config.ts` 中 `server.port` |
| 端口 8000 被占用 | 其他进程 | `uvicorn ... --port 8001`，同步修改 `vite.config.ts` 中的代理目标 |
